# fontplay

## Project Description
Upload a font file (.ttf / .otf / .woff2), type letters, and see them rendered live using the
uploaded font. Choose a mood — the AI picks a combination of geometric transforms and COLRv1
color effects. Rust/WASM rewrites every glyph outline and builds a new font file in the browser
(powered by [fontations/skrifa](https://github.com/googlefonts/fontations) + write-fonts).
The styled font is injected via `@font-face` and can be downloaded as a TTF with the mood baked
into the name table and both COLRv0 + COLRv1 color tables for broad app compatibility.

## Stack
- Vue 3 (Composition API, `<script setup lang="ts">`)
- TypeScript (strict)
- Vite 8
- Rust + [skrifa](https://docs.rs/skrifa) compiled to WASM via `wasm-pack`
- Ollama locally (model: qwen2.5-coder)
- Anthropic SDK via Netlify Function (production, model: claude-sonnet-4-6)
- Biome (formatter + linter)
- CSS Reset: @csstools/normalize.css via CSS Layers

## WASM Build
The Rust crate is in `rust/` and outputs to `src/wasm/` (gitignore this folder).

```bash
# First time / after changing Rust code:
wasm-pack build rust --target web --out-dir ../src/wasm --out-name fontplay
```

The WASM module is lazy-loaded in `useFontWasm.ts` on first font upload.

**Prerequisites:**
```bash
brew install rustup && rustup-init
rustup target add wasm32-unknown-unknown
brew install wasm-pack
```

## WASM API (rust/src/lib.rs)
| Export | Input | Returns |
|---|---|---|
| `get_font_info(data: &[u8])` | Raw font bytes (TTF/OTF/WOFF2) | `{ unitsPerEm, ascender, descender, glyphCount }` |
| `style_font(data: &[u8], request_json: &str)` | Font bytes + JSON request | Styled TTF bytes |

`style_font` request shape:
```json
{ "transforms": [...], "colr": { "effects": [...], "fillColor": "...", ... }, "mood": "playful" }
```

WOFF2 inputs are decoded to SFNT via `woff2-no-std`. WOFF1 is rejected with an error message.
All glyph outlines are re-written from scratch using write-fonts / kurbo. The output is always
a TrueType (glyf-based) font regardless of input format.

## LLM Abstraction
All AI calls go through `askLLM(property)` in `useLLM.ts`:

- `VITE_OLLAMA_URL` is set → calls Ollama directly from the browser
- `VITE_OLLAMA_URL` is not set → calls the Netlify Function at `/api/ask`

## AI Response Format
The LLM returns a JSON object with transforms and visual effects:
```json
{
  "transforms": [{"type": "scaleX", "factor": 0.8}, {"type": "shear", "angle": -10}],
  "effects": {
    "active": ["shadow", "fill"],
    "fillColor": "#e63946",
    "outlineColor": "#2563eb",
    "blockColor": "#111111",
    "gradientColors": ["#f97316", "#8b5cf6"]
  }
}
```
Available transform types: `scaleX`, `scaleY`, `shear`, `jitter`, `wave` — defined in `shared/types.ts`.

Available effects: `shadow`, `3d-blocks`, `outline`, `fill`, `gradient` — rendered as SVG paint
layers in `GlyphDisplay.vue` (COLRv1-inspired). `fill` and `gradient` are mutually exclusive.

`extractResult()` in `shared/utils/extractSvg.ts` parses the raw LLM output, with fallback
recovery for truncated responses.

Transforms are applied in Rust to every glyph outline in the font. The original font bytes are
always preserved in memory — each mood change re-runs `style_font` from the original.

## CSS Architecture
- Layer order: `reset → base → utilities`
- Component-specific styles live in `<style scoped>` inside each `.vue` file
- Only globally reusable classes (.btn, .sr-only, .error, .loading, .info) go in `utilities.css`
- CSS custom properties are declared in `:root` inside `base.css`

## Netlify Deployment
- Build command installs `wasm-pack` via the init script, then runs `npm run build`
- Set `ANTHROPIC_API_KEY` in the Netlify dashboard
- `VITE_OLLAMA_URL` must NOT be set in production (triggers Netlify Function path)
- Rate limiting: Netlify Blobs store (`sessions`) tracks requests per session ID
