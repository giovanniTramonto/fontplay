# fontplay

## Project Description
Upload a font file (.ttf / .otf / .woff2), type letters, and see them rendered live using the
uploaded font. Three modes are available:

- **Mood** — the AI picks geometric transforms and COLRv1 color effects; Rust/WASM rewrites every glyph outline and builds a new font file in the browser.
- **Blend** — two fonts are rasterised to canvas bitmaps per glyph, blended via SDF morphing in WASM, and written back as a new TTF.
- **Recombine** — Claude Vision (or a local Ollama vision model) designs a hybrid SVG glyph from two font renderings; the SVG path is parsed and injected into the font via WASM.

The result is injected via `@font-face` and can be downloaded as a TTF.

## Stack
- Vue 3 (Composition API, `<script setup lang="ts">`)
- TypeScript (strict)
- Vite 8
- Rust + [skrifa](https://docs.rs/skrifa) compiled to WASM via `wasm-pack`
- Ollama locally (model configured via `VITE_OLLAMA_MODEL`, must be vision-capable for Recombine e.g. `llava`)
- Anthropic SDK via Netlify Functions (production)
- Biome (formatter + linter)
- CSS Reset: @csstools/normalize.css via CSS Layers

## WASM Build
The Rust crate is in `rust/` and outputs to `src/wasm/` (gitignore this folder).

```bash
# First time / after changing Rust code:
wasm-pack build rust --target web --out-dir ../src/wasm --out-name fontplay
```

The WASM module is lazy-loaded in `useFontWasm.ts` on first font upload.

**Prerequisites:**
```bash
brew install rustup && rustup-init
rustup target add wasm32-unknown-unknown
brew install wasm-pack
```

## WASM API (rust/src/lib.rs)
| Export | Input | Returns |
|---|---|---|
| `get_font_info(data: &[u8])` | Raw font bytes (TTF/OTF/WOFF2) | `{ unitsPerEm, ascender, descender, glyphCount }` |
| `style_font(data: &[u8], request_json: &str)` | Font bytes + JSON request | Styled TTF bytes |
| `blend_from_canvas_bitmaps(font1, font2, bitmaps1, bitmaps2, request_json)` | Two fonts + grayscale canvas bitmaps | Blended TTF bytes |
| `recombine_fonts_with_paths(font1, request_json)` | Font bytes + `{ charPaths: { [char]: svgPath } }` | Recombined TTF bytes |

`style_font` request shape:
```json
{ "transforms": [...], "colr": { "effects": [...], "fillColor": "...", "outlineColor": "...", "gradientColors": [...], "blockColor": "..." }, "mood": "playful" }
```

WOFF2 inputs are decoded to SFNT via `woff2-no-std`. WOFF1 is rejected with an error message.
All glyph outlines are re-written from scratch using write-fonts / kurbo. The output is always
a TrueType (glyf-based) font regardless of input format.

## LLM Abstraction (`useLLM.ts`)
- `VITE_OLLAMA_URL` is set → calls Ollama directly from the browser
- `VITE_OLLAMA_URL` is not set → calls the corresponding Netlify Function

| Function | Local (Ollama) | Production (Netlify) |
|---|---|---|
| `askLLM(property)` | `/api/chat` with `VITE_OLLAMA_MODEL` | `/api/ask` (claude-sonnet-4-6) |
| `askRecombineLLM(char, img1, img2)` | `/api/chat` with vision model | `/api/recombine` (claude-opus-4-5) |

## AI Response Format (Mood)
```json
{
  "transforms": [{"type": "scaleX", "factor": 0.8}, {"type": "shear", "angle": -10}],
  "effects": {
    "active": ["shadow", "fill"],
    "fillColor": "#e63946",
    "outlineColor": "#2563eb",
    "blockColor": "#111111",
    "gradientColors": ["#f97316", "#8b5cf6", "#1e1b4b"]
  }
}
```
Available transforms: `scaleX`, `scaleY`, `shear`, `jitter`, `wave`, `waveY`, `rotate`, `perspective`, `arch`

Available effects: `shadow`, `3d-blocks`, `outline`, `fill`, `gradient`, `highlight`

`parseLLMResult.ts` parses raw LLM output with fallback recovery. Unknown transform types are filtered before passing to WASM.

## Ollama Configuration (local)
- Base URL: `http://localhost:11434/api/chat`
- Model: set `VITE_OLLAMA_MODEL` — use a vision-capable model (e.g. `llava`) for all three features
- Copy `.env.example` → `.env` and adjust if needed.

## CSS Architecture
- Layer order: `reset → base → utilities`
- Component-specific styles live in `<style scoped>` inside each `.vue` file
- Only globally reusable classes (.btn, .sr-only, .error, .loading, .text-size-m) go in `utilities.css`
- CSS custom properties are declared in `:root` inside `base.css`

## Netlify Deployment
- Build command installs `wasm-pack` via the init script, then runs `npm run build`
- Set `ANTHROPIC_API_KEY` in the Netlify dashboard
- `VITE_OLLAMA_URL` must NOT be set in production (triggers Netlify Function path)
- Rate limiting: Netlify Blobs store (`sessions`) tracks requests per session ID
